### Key Docker Concepts:
 - <kbd style='color:#cbb786'>**Images**:</kbd> Docker images are the building blocks of containers. They contain everything needed to run an application, including the code, runtime, system tools, and libraries.
 - <kbd style='color:#cbb786'>**Containers**:</kbd> Containers are instances of Docker images. They run in isolated environments and provide a consistent runtime environment for applications.
 - <kbd style='color:#cbb786'>**Docker Hub**:</kbd> Docker Hub is a repository for finding and sharing Docker images.

## Essential Docker Commands

- <kbd style='color:#cbb786'>**docker build**</kbd> - Build an image from a Dockerfile
- <kbd style='color:#cbb786'>**docker run**</kbd> - Create and run a new container from an image
- <kbd style='color:#cbb786'>**docker ps**</kbd> - List containers
- <kbd style='color:#cbb786'>**docker start**</kbd> - Start one or more stopped containers
- <kbd style='color:#cbb786'>**docker stop**</kbd> - Stop one or more running containers
- <kbd style='color:#cbb786'>**docker rm**</kbd> - Remove one or more containers
- <kbd style='color:#cbb786'>**docker rmi**</kbd> - Remove one or more images
- <kbd style='color:#cbb786'>**docker image**</kbd> - Manage images
- <kbd style='color:#cbb786'>**docker images**</kbd> - Lists all images stored locally
- <kbd style='color:#cbb786'>**docker pull**</kbd> - Downloads an image or a repository from a registry
- <kbd style='color:#cbb786'>**docker push**</kbd> - Uploads an image or a repository to a registry
- <kbd style='color:#cbb786'>**docker login**</kbd> - Authenticates to a Docker registry
- <kbd style='color:#cbb786'>**docker logout**</kbd> - Logs out from a Docker registry
- <kbd style='color:#cbb786'>**docker exec**</kbd> - Runs a command in a running container
- <kbd style='color:#cbb786'>**docker logs**</kbd> - Fetches the logs of a container
- <kbd style='color:#cbb786'>**docker network**</kbd> - Manages networks
- <kbd style='color:#cbb786'>**docker volume**</kbd> - Manages volumes. Volumes are used to persist data generated by and used by Docker containers

---

 ### `docker build`
 Builds a Docker image from a Dockerfile. A Dockerfile contains a set of instructions for creating an image that includes the application, its dependencies, and runtime environments.

 <kbd style='color:#cbb786'>**docker build [OPTIONS] PATH | URL**</kbd>
 - <kbd style='color:#cbb786'>PATH</kbd> specifies the location of the Dockerfile and related context (e.g., files used in the build).
 - <kbd style='color:#cbb786'>URL</kbd> can be used to specify a Git repository as the source of the Dockerfile and context.
 - **Optional parameters**
   - <kbd style='color:#cbb786'>-t, --tag</kbd>: Name and optionally a tag in the 'name:tag' format.
   - <kbd style='color:#cbb786'>--build-arg</kbd> Set build-time variables.
   - <kbd style='color:#cbb786'>--no-cache</kbd> Do not use cache when building the image.
   - <kbd style='color:#cbb786'>-f, --file</kbd> Name of the Dockerfile (Default is PATH/Dockerfile).

**Examples:**
 - <kbd style='color:#cbb786'>docker build -t {image name} -f Dockerfile .</kbd> - build image with name.
 - <kbd style='color:#cbb786'>docker build -t {image name}:{tag} --no-cache --build-arg {arguments like - ENVIRONMENT=production} -f Dockerfile .</kbd> - build image with name, arguments and no cache during building.

---

### `docker run`
 Creates a new container from a specified image and starts it. If the image is not available locally, Docker will attempt to pull it from the configured registry.

<kbd style='color:#cbb786'>**docker run [OPTIONS] IMAGE [COMMAND] [ARG...]**</kbd>
 - <kbd style='color:#cbb786'>IMAGE</kbd> is the name of the image to start the container from.
 - <kbd style='color:#cbb786'>COMMAND</kbd> and ARG specify the command to run inside the container.
 - **Optional parameters**
   - <kbd style='color:#cbb786'>-d, --detach</kbd> Run container in background and print container ID.
   - <kbd style='color:#cbb786'>--name</kbd> Assign a custom name to the container. If not specified, Docker generates a random name.
   - <kbd style='color:#cbb786'>-p, --publish</kbd> Publish a container's port(s) to the host. Format: hostPort:containerPort.
   - <kbd style='color:#cbb786'>-e, --env</kbd> Set environment variables. Format: VARIABLE=value.
   - <kbd style='color:#cbb786'>--rm</kbd> Automatically remove the container when it exits. This is useful for not leaving behind any stopped containers.
   - <kbd style='color:#cbb786'>-v, --volume</kbd> Mount a volume. Format: hostSrc:containerDest or just the volume name if using Docker volumes.

**Examples:**
 - <kbd style='color:#cbb786'>docker run --name {container name} {image name}</kbd> - start a container from an image.
 - <kbd style='color:#cbb786'>docker run -d --name {container name} {image name}</kbd> - start a container in the background.
 - <kbd style='color:#cbb786'>docker run -e ENVIRONMENT=production --name {container name} {image name}</kbd> - start a container with an environment variable.

---

 ### `docker ps`
 Lists containers. By default, it shows only running containers.

<kbd style='color:#cbb786'>**docker ps [OPTIONS]**</kbd>

**Optional parameters**
 - <kbd style='color:#cbb786'>-a, --all</kbd>: Show all containers (default shows just running).
 - <kbd style='color:#cbb786'>-q, --quiet</kbd>: Only display container IDs.
 - <kbd style='color:#cbb786'>-f, --filter</kbd>: Filter the output based on conditions like status, id, name, etc.

**Examples:**
 - <kbd style='color:#cbb786'>docker ps</kbd> - List running containers
 - <kbd style='color:#cbb786'>docker ps -a</kbd> - List all containers, including those that are stopped
 - <kbd style='color:#cbb786'>docker ps -q</kbd> - List the IDs of all running containers
